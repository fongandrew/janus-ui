List rhythm
Sets the cascading --v-list-rhythm token. Grouped = discrete points (marketing). Continuous = line-height − 1cap, so wrapped bullets read as one passage (WYSIWYG). Watch the multi-line bullets above.
Header height (o-bar §9.7)
Contains = reduced block padding so the control's text sits --v-spacing from the edge (fixes the too-tall header). Input = bar is exactly one control tall. Text = full box padding.
Inset · global (form labels)
How far a field's label + description indent to sit over the control below them (the v1 look). Drag freely, or snap: 0 = the control's outer edge, Radius = its straight edge past the corner (the control's own radius), Text = the control's text. The control stays on the edge; only the label/description move.
Inset · prose (wide width)
A container with boxes insets its direct prose children to meet the cards: 0 = the cards' outer edge, Radius = their straight edge (--o-box__radius), Text = their inner text (--v-pad). Block boxes (code, blockquotes) break out. A toggle, not a slider — because at the full-bleed breakpoint the target has to switch (see below), not slide. The dialog uses the same idea but defaults to just Radius.
Full-bleed boxes (narrow frames)
One toggle for every frame. Full-bleed isn't a box property — it's the frame → box → control structure adapting. On: in any frame narrower than the breakpoint (a @container), boxes break out to the frame edges and drop their radius/borders — so the page's cards on a narrow viewport and a dialog's box when the dialog is narrow both go full-bleed by the same rule. Prose flushes to the boxes' inner text. Drag the preview narrow (and open the dialog) to see both.

Heading & title groups
Space above heading  ×line
A bare heading owns only the space above it; below it the following element keeps its ③ prose gap.
Title-group leading  ×spacing
Constant space between every line in an <hgroup> — line-height = 1em + this, and the title↔subtitle gap is the natural half-leading (no margin, trim only at the outer edges), so it matches the spacing between lines within the title and subtitle. Narrow the preview to wrap the title.
Radius & nesting (§8)
Quick presets (just set min/max)
One always-on cascade — no preset classes. The window holds the max radius; everything inside steps down toward the min by the padding between it and its parent (cards = window − gutter, controls a step further if nested, dialog = window − inset). These buttons just set the two sliders below: Stepped = large range, Near-uniform = small range (inner levels bottom out at the floor), Flat = min == max.
Max radius · the window
The outermost radius, carried by the window frame. Everything inside steps down from here. (Large values make the inward steps visible before they bottom out at the min.)
Min radius · the floor
Nothing rounds below this, so corners never go sharp — deep/small elements just bottom out here.
Dialog inset · --o-dialog__offset  ×spacing
How far the dialog sits inside the window. dialog radius = window − this, so a bigger inset rounds the dialog less.
Draw window radius
Whether the preview window actually paints the max radius. Off = we don't draw it (the browser/OS rounds the window) — but the cascade still steps down from the expected max, so the cards stay sized for it. The radius is a value we assume, separate from whether we paint it.
Dialog (§ modal — folded forks)
Header treatment
Open the dialog (the Open dialog button in the card grid) and scroll: the header slides away, only the X stays pinned. Plain = surface header sharing the body's --v-pad seam. Glass = tinted two-tone panel; being two-tone it can't reuse that seam, so it gets extra padding before the body.
Sticky footer
Opt-in stickiness. None = the footer scrolls with the content. Submit = only the primary Save pins (bottom-right); secondary actions scroll away. Whole bar = the entire footer pins as a glass action bar. Only visible once the dialog overflows.
